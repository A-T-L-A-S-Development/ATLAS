# Version 1.0 Release (Official)

## TL;DR
After over a year of development, ATLAS Development is releasing Version 1.0 as a full product hub, licensing backend, store dashboard, support system, and staff operations console. V1 improves the seller-buyer workflow, hardens delivery and authentication, expands internal automation, and gives customers a faster, clearer web experience.

This release is production-ready for broader use across customers, sellers, support staff, and ATLAS operators.

## Release Notes (Short)
- Launched the full ATLAS store experience with login, account overview, product browsing, key management, support, credits, Blackjack, and audio access flows.
- Expanded product delivery with product codes, no-license-key products, protected download paths, catalog pagination, grouped product files, and visual key cards.
- Added Discord-backed support tickets plus Discord slash commands for opening, closing, retrieving profiles, saving PDF transcripts, and deleting closed ticket channels.
- Added ATLAS Nexus Discord staff commands for account linking, account lookup, profile inspection, unlinking, role sync, role-binding summaries, and account audits.
- Added reward invite generation, regeneration, revocation, redemption, Discord guild membership checks, credit rewards, and admin invite controls.
- Added stronger Lua obfuscation and protected loadstring delivery for eligible products.
- Added Heimdal public pages and affiliate information.
- Strengthened authentication, OAuth, 2FA, Turnstile, SRI, CORS, CSRF, redirect, proxy-header, and sensitive endpoint protections.

## Full Release Notes

### Store, Accounts, and Customer Experience
- Redesigned the store hub with dedicated pages for overview, products, keys, account security, support, Blackjack, admin tools, and audio access.
- Added Roblox OAuth login, GitHub OAuth sign-in/linking, top-window-aware OAuth navigation, and improved GitHub OAuth callback handling.
- Added account security management with permanent passwords, temporary password migration, TOTP, passkeys, preferred 2FA method selection, passkey quick login, and GitHub unlinking.
- Added Cloudflare Turnstile enforcement for store password login, with the Turnstile SRI hash injected from backend configuration.
- Added product browsing with featured products, ownership state, credit and USD purchase paths, support-ticket handoff for USD purchases, product details, gallery support, and ratings.
- Added product ratings and review aggregation so owned users can rate products and product cards show average ratings and counts.
- Added a user overview dashboard with account summary, Roblox and Discord details, credit balance, announcements, product counts, activity, favorite audios, and reward invite status.
- Added key library improvements including searchable product cards, product visuals, watermarked product key cards, bind previews, key rebind flows, detached download handling, and no-license-key product support.
- Added no-license-key products across catalog grants, admin license grants, key responses, and product download UI.
- Added protected download paths for product assets so downloads are routed through validated product and account context.
- Added public release note loading and rendering through versioned release-note links.

### Credits, Rewards, Blackjack, and Audio Access
- Added a credit wallet with balance, purchase, add, subtract, edit, recent purchase, and admin adjustment flows.
- Added Blackjack gameplay backed by credits, including deal, hit, stand, double, split, next-hand, state, wager logs, and admin log deletion.
- Added reward invites with one active invite per account, generate/regenerate/revoke actions, invite landing pages, Discord guild membership verification, atomic redemption protection, referral tracking, and fixed credit rewards.
- Added admin reward invite visibility and revoke controls in the account inspector.
- Added the ATLAS Audio Library with categories, favorites, copy tracking, flagging, paywall/access purchase flows, lifetime access handling, access metrics, and admin metrics reset.

### Support and Discord Operations
- Added web support tickets with authenticated ticket creation, ticket lists, ticket detail, replies, product/license pickers, issue metadata, Discord channel creation, message sync, and closed-ticket handling.
- Added support account lookup and suggestions by Roblox ID, Discord ID, username context, or license lookup where authorized.
- Added ticket resolution controls for support and product audit workflows.
- Added support-ticket Discord slash commands:
  - `/open-ticket` for linked web-backed tickets and Discord-only tickets.
  - `/retrieve-profile` for support/admin profile retrieval inside ticket channels.
  - `/save-transcript` for admin PDF transcript generation.
  - `/delete` for admin deletion of already-closed ticket channels and matching web records.
- Added ATLAS Nexus Discord staff commands:
  - `/admin link-accounts` with modal input, conflict preview, and confirmation.
  - `/admin user-profile` for read-only profile inspection.
  - `/admin unlink-account` with confirmation while preserving the account record.
  - `/admin force-update` for forced role/nickname sync.
  - `/admin role-bindings` for configured Roblox rank role summaries.
  - `/admin account-audit` for metadata and account flags without raw secrets.
  - `/support account-lookup` for read-only staff account lookup.
- Kept Discord command responses ephemeral where staff/customer data is involved and enforced Discord admin/support role checks server-side.

### Admin Console and Staff Tooling
- Added richer admin account inspection with licenses, credits, Blackjack logs, passkey/TOTP state, security flags, sessions, reward invites, account metadata, and recent activity.
- Added admin account actions for license add/update/remove/refund/revoke-all, credit add/subtract/edit/settings, password updates, permission updates, link reset, security clear, session clear, and Blackjack log deletion.
- Added recent accounts endpoint and UI for faster staff account triage.
- Added admin role binding management for Roblox rank to Discord role synchronization.
- Added announcement create/delete flows used by store dashboards.
- Added product catalog admin APIs, catalog asset image handling, product folder catalog layout support, paginated catalog responses, and client sync improvements.
- Added audit log viewing, issue resolution, product audit deletion, and safer audit redaction.
- Added avatar cache clearing, MongoDB size checks, staff data controls, and stronger API documentation through OpenAPI.

### Obfuscation, Loadstring Protection, and Developer Tooling
- Rebuilt the Lua obfuscator with stronger VM-based protection, configurable packing controls, better output randomization, and improved runtime hardening.
- Removed unsafe fallback behavior from normal obfuscation output.
- Added whitelist V3 support with safer license-based authentication.
- Added protected loadstring delivery for eligible whitelist products.
- Added trace/debug controls for protected loader diagnostics while keeping production error redaction enabled by default.
- Added protected version management, improved admin workflows, task checklist UI, and bounded rendering for very large generated loaders.

### Heimdal
- Added Heimdal public pages and affiliate information for the ATLAS site.

### Security, Reliability, and Platform Hardening
- Added encrypted storage and lookup hashing for sensitive account fields and license secrets.
- Added password hashing, temporary password migration, passkey normalization, TOTP challenge handling, session token hashing, and store session trimming.
- Added hardened bearer checks for sensitive endpoints, including license-key validation with constant-time bearer/body comparison.
- Added stricter redirect handling, release-note URL allowlisting, proxy header checks, and Cloudflare proxy requirements.
- Added CORS lockdown, CSRF validation, secure asset navigation checks, path traversal protections, sanitized filenames, sanitized release-note HTML, and safer Express responses.
- Added Subresource Integrity enforcement for external non-image resources, including the script to generate/update SRI hashes.
- Added safer outbound request handling with issue logging, allowlisted paths, generic error responses, and no raw secret logging.
- Added structured audit and Discord logging for account, product, support, and validation flows while redacting sensitive values.

### API and Documentation
- Updated `JsonFiles/openapi.json` to document the current HTTP API surface, including store, support, reward invite, audio, admin, obfuscation, and delivery routes.
- Added tests covering reward invites, support bot commands, Discord staff commands, sensitive endpoint hardening, loadstring protection, obfuscator hardening, store key downloads, getkeys responses, security controls, and Turnstile integrity.

## What Changed for Users and Developers
- Users get a clearer store dashboard, product library, key downloads, support tickets, reward invites, credits, audio access, stronger account security, and faster delivery.
- Sellers get better product catalog handling, safer key delivery, no-license-key product support, ratings, product visuals, and protected download flows.
- Support staff get web and Discord ticket workflows, account lookup, profile retrieval, PDF transcripts, issue resolution, and Discord-only ticket support.
- Admins get richer account controls, audits, recent accounts, role bindings, reward invite controls, protected obfuscation tooling, and Discord-based account operations.
- Developers/operators get a stricter OpenAPI contract, broader automated tests, hardened endpoints, stronger observability, and safer product/obfuscation workflows.

## Known Limitations and Roadmap
- Real-currency payment storefront automation remains planned for a later major version because of security, payment, compliance, and tax complexity.
- Ongoing improvements include expanded fraud automation, deeper analytics, more developer-facing tooling, and continued hardening of internal staff workflows.
